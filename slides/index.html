INTRO

WHAT IT REALLY MEANS

바이브코딩은 코딩만 하는 일이 아닙니다

내 일을 더 쉽게, 더 빠르게 만든 뒤, 점점 더 큰 덩어리까지 AI에 맡기는 방식입니다.

일을 쉽게 속도를 빠르게 덩어리째 위임
자동화와 에이전트는 AI가 우리 대신 일을 하기 시작한 형태입니다.
지금 우리가 배우는 것은 코딩 문법보다 AI에게 일을 배정하는 법입니다.

1. 먼저 내 속도를 붙입니다

정리, 요약, 초안, 반복 작업부터 더 가볍고 빠르게 처리합니다.

2. 그다음 큰 조각을 맡깁니다

한 문서, 한 기능, 한 흐름처럼 사람이 하던 묶음을 통째로 넘기기 시작합니다.

3. 실행 형태가 자동화와 에이전트입니다

AI가 이어서 실행하고, 정리하고, 다음 단계를 준비하는 방식으로 발전합니다.

MODULE 2 | HANDS-ON

실습 1: 회의 메모 / 프로젝트 업데이트 정리기 (5분)

기본 프롬프트

회의 메모 / 프로젝트 업데이트 정리기를 만들어줘.
먼저 상황을 정리하고,
내가 결정해야 할 것을 3~5개 질문하고,
답을 반영한 5줄 계획을 보여준 뒤,
그 다음 가벼운 결과 화면을 만들어줘.
- 입력: 회의 메모 또는 프로젝트 업데이트 텍스트
- 출력: 대상별 요약, 다음 액션 3개, 결정사항, 공유용 한 줄 업데이트
- 모바일 대응
- 한 화면에 가볍게

변형 아이디어

  • 경영진용 / 실무자용 요약 전환
  • “꼭 읽을 것” 강조 영역 추가
  • 결정사항 로그 강조
  • 공유용 한 줄 업데이트 복사 버튼

질문이 먼저 나오면 잘 되고 있는 겁니다. 답을 준 뒤 5줄 계획과 화면을 받으세요.

물 한 잔, 스트레칭, 질문 메모. 다음 세션부터는 대상·채널·톤 같은 변수를 정리하며 실제 도구를 만듭니다.

MODULE 3

Agent 심화: 로컬 Agent + Subagents

도구역할언제 쓰나
Chat설명/질문흐름 이해, 비교, 요약
Agent편집+실행도구 생성, 반복 수정
Plan Mode구현 전 설계범위가 큰 작업
Subagents병렬 탐색/실행긴 탐색, noisy 작업

내장 Subagents

  • Explore — 코드베이스 탐색
  • Bash — 긴 셸 출력 격리
  • Browser — 브라우저 상호작용 요약

왜 중요한가?

탐색/로그/브라우저 같은 noisy 작업을 메인 대화 밖으로 빼서, Agent가 더 또렷한 결정을 내리게 합니다.
