# 참가자 퀵스타트 — 나만의 바이브코딩 4단계

## 오늘의 목표
- Cursor를 설치하고 기본 단축키를 익힌다.
- **리서치 → 계획 → 구현 → 검토** 흐름을 직접 한 번 끝까지 돌린다.
- 다음 주에 바로 써볼 문장 1개를 챙긴다.

## 오늘 바로 쓰는 4문장
1. **리서치** — “먼저 상황을 정리해줘”
2. **계획** — “시작 전에 5줄 계획을 보여줘”
3. **구현** — “이 범위만 만들어줘”
4. **검토** — “바뀐 점과 확인 방법을 알려줘”

## 오늘 자주 쓰는 단축키
| 단축키 | 기능 |
|--------|------|
| `Cmd+I` / `Ctrl+I` | Agent |
| `Cmd+L` / `Ctrl+L` | Chat |
| `Cmd+K` / `Ctrl+K` | Inline Edit |
| `Shift+Tab` | Plan Mode |
| `Tab` | 제안 수락 |

## 실습 1 목표
- 회의 메모 / 프로젝트 업데이트 정리기를 만든다.
- Plan Mode가 먼저 질문하고 5줄 계획으로 좁히는 흐름을 본다.

## 실습 2 목표
- 공지/FAQ 생성기를 만든다.
- 대상, 채널, 톤을 질문으로 정리한 뒤 구현하는 흐름을 경험한다.

## 실습 3 목표
- 작은 추가와 짧은 수정을 해본다.
- 마지막에 확인 방법까지 남긴다.

## 막힐 때 바로 쓰는 문장
```text
내가 원하는 결과를 아직 정확히 정리하지 못했어.
먼저 상황을 정리하고, 내가 결정해야 할 것을 질문해줘.
답을 반영한 5줄 계획도 보여줘.
```

## 오늘 지킬 원칙
- 실제 고객정보, 비밀번호, 키는 넣지 않는다.
- 외부 API 대신 mock 데이터를 사용한다.
- 한 번에 크게 시키지 말고 작은 범위로 나눈다.
- 결과를 받으면 바로 넘기지 말고 검토 문장을 붙인다.
