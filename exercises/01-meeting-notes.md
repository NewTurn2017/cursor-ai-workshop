# 실습 1: 회의 메모 / 프로젝트 업데이트 정리기

> **4단계 중**: 리서치 + 계획 + 구현
> **소요 시간**: 약 10분
> **사용 기능**: Agent (Cmd+I), Plan Mode (Shift+Tab), Chat (Cmd+L)

## 목표

- Plan Mode가 먼저 질문하고, 5줄 계획으로 좁히는 흐름을 경험한다.
- Tab, Chat, Inline Edit를 **리서치 도구**로 써본다.

## Step 1: 프롬프트 입력

Agent (Cmd+I)를 열고 아래 프롬프트를 입력합니다:

```text
회의 메모 / 프로젝트 업데이트 정리기를 만들어줘.
먼저 상황을 정리하고,
내가 결정해야 할 것을 3~5개 질문하고,
답을 반영한 5줄 계획을 보여준 뒤,
그 다음 가벼운 결과 화면을 만들어줘.
- 입력: 회의 메모 또는 프로젝트 업데이트 텍스트
- 출력: 대상별 요약, 다음 액션 3개, 결정사항, 공유용 한 줄 업데이트
- 모바일 대응
- 한 화면에 가볍게
```

## Step 2: 질문에 답하기

AI가 질문을 하면 자유롭게 답합니다. 예시:
- 대상: "팀 내부용"
- 형식: "텍스트 붙여넣기"
- 출력 톤: "간결하게"

## Step 3: 결과 확인 + 검토

결과를 받은 후 아래 문장으로 검토합니다:

```text
이 화면 구조와 사용 흐름을 3문장으로 설명해줘.
```

## 코칭 포인트

- 질문이 먼저 나오면 잘 되고 있는 것
- 결과가 길면: "한 화면 안에 들어오게 줄여줘"
- 막히면: "먼저 상황을 정리해줘"부터 시작

## 성공 기준

- [ ] AI가 먼저 질문을 했다
- [ ] 5줄 계획을 확인했다
- [ ] 결과물이 한 화면에 들어온다
- [ ] 검토 문장을 1개 이상 붙였다
