# 실습 2: 공지/FAQ 생성기

> **4단계 중**: 계획 + 구현
> **소요 시간**: 약 20분
> **사용 기능**: Agent (Cmd+I), Plan Mode (Shift+Tab)

## 목표

- Agent와 Plan Mode를 **계획 → 구현** 흐름으로 연결한다.
- 멀티파일 작업에서도 작은 범위를 유지하는 법을 익힌다.

## Step 1: 프롬프트 입력

```text
공지/FAQ 생성기를 만들어줘.
먼저 아래 정보 중 빠진 것을 질문해줘:
- 대상
- 채널
- 톤
- 필수 포함 항목
- FAQ 개수
- 입력 형식

답을 받은 뒤 5줄 계획을 보여주고,
그 다음 가벼운 결과 화면을 만들어줘.
기능: 공지 제목/본문 초안, FAQ 3~5개, 꼭 읽어야 할 핵심 요약, 복사하기 쉬운 출력 블록
```

## Step 2: 질문 답변 예시

- 대상: "신규 입사자"
- 채널: "사내 메신저"
- 톤: "친근하지만 정확하게"
- 필수 포함: "날짜, 장소, 준비물"
- FAQ 개수: 5개
- 입력: "텍스트 직접 입력"

## Step 3: 결과 확인

- Plan Mode가 5줄 계획을 보여주는지 확인
- 파일이 생성되면 **diff를 읽기** (바뀐 부분 확인)
- 실행해서 동작 확인

## Step 4: 기능 추가 (2차 프롬프트)

```text
이 생성기를 더 쓰기 좋게 다듬어줘.
- 대상별 톤 전환
- 꼭 읽어야 할 안내 박스 추가
- FAQ 개수 선택 UI 추가
```

## 컨텍스트 운영 원칙

- **한 대화 = 한 기능** 또는 한 수정
- 범위가 바뀌면 **새 대화**를 연다
- 이전 내용은 **요약만** 다시 넣는다

## 성공 기준

- [ ] 질문 → 계획 → 구현 흐름이 끊기지 않았다
- [ ] 5줄 계획을 확인한 뒤 구현으로 넘어갔다
- [ ] diff를 한 번 이상 확인했다
- [ ] 2차 수정에서 작은 범위만 요청했다
