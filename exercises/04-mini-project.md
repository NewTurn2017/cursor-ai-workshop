# 실습 4: 4단계 미니 프로젝트

> **4단계 전체**: 리서치 → 계획 → 구현 → 검토
> **소요 시간**: 약 25분
> **목표**: 하나를 끝까지 돌려보기

## 목표

- 4단계를 **처음부터 끝까지** 한 번 돌린다.
- 각자 바로 떠올릴 수 있는 작은 도구 하나를 완성한다.

## 프로젝트 옵션 (하나를 고르세요)

### A. 회의 메모 정리기
메모를 요약, 결정사항, 다음 액션으로 정리하는 도구

### B. 공지/FAQ 생성기
입력값으로 공지문과 FAQ 초안을 생성하는 도구

### C. 고객 문의 분류기
문의를 유형, 우선순위, 답변 초안으로 분류하는 도구

### D. 주간 업데이트 카드
한 주 진행상황을 공유용 카드 한 장으로 정리하는 도구

## 진행 순서

### 1단계: 리서치 (3분)

```text
[선택한 도구]를 만들려고 해.
먼저 상황을 정리하고, 내가 결정해야 할 것을 질문해줘.
```

### 2단계: 계획 (3분)

```text
답변을 반영해서 5줄 계획을 보여줘.
```

### 3단계: 구현 (15분)

```text
이 계획대로 만들어줘. 이 범위만 먼저 만들어줘.
```

- 한 번에 전부 말고, **한 범위씩**
- 막히면 계획으로 돌아간다

### 4단계: 검토 (4분)

```text
바뀐 점과 확인 방법을 알려줘.
```

- diff 확인
- 실행 확인
- 검토 문장 남기기

## 막힐 때

```text
내가 원하는 결과를 아직 정확히 정리하지 못했어.
먼저 상황을 정리하고, 내가 결정해야 할 것을 질문해줘.
답을 반영한 5줄 계획도 보여줘.
```

## 성공 기준

- [ ] 리서치 문장을 먼저 썼다
- [ ] 5줄 계획을 확인했다
- [ ] 한 범위씩 구현했다
- [ ] 검토 문장으로 마무리했다
- [ ] 동작하는 결과물이 하나 있다
